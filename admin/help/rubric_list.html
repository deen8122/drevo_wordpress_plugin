Рекомендации по созданию и ведению рубрикатора.

  1. Рубрика (товарная группа) предназначена для обозначения множества товаров, принадлежащих к этой группе (напр. погрузчики, компьютеры, телевизоры) с одинаковой структурой характеристик (напр. страна-производитель, ширина, вес, и т.д.).
    Имя рубрики обязательно должно включать имя существительное. Если имени существительного нет, обратитесь к администратору системы для получения консультации.
  2. Подрубрики создаются для разделения на подгруппы товаров в группе:
    • разные наименования товаров (напр. если в группе «бытовая техника» содержатся пылесосы и утюги, то создайте аналогичные подрубрики)
    • если товары имеют отличающуюся структуру характеристик (напр. в товарной группе "Телевизоры" могут быть ЖК-телевизоры, и плазменные. У ЖК-телевизоров есть характеристика "время отклика", а у плазменных её нет, но есть другие характеристики. В этом случае необходимо создать 2 подрубрики "ЖК" и "Плазменные")
  3. Если же логически требуется разделить рубрику на подрубрики, и все товары всех подрубрик имеют одинаковую структуру характеристик, создавать подрубрики не требуется, а рекомендуется создать характеристику – справочник значений, и отметить её, как градацию на сайте. (напр. рубрику "Флеш-накопители" логически можно разделить на подгруппы по объемам памяти (128 Мб, 256 Мб, 1Гб, 8 Гб), но флеш-накопители любых объёмов не имеют различной структуры характеристик. В этом случае создавать подрубрик не рекомендуется.)
  4. Градация – это свойство характеристики, которое отображает важность характеристики перед другими характеристиками. Это свойство могут принимать характеристики только с типом данных «справочник». По градациям товарные позиции будут объединяться на сайте, для более удобной навигации посетителей.
  5. Не рекомендуется создавать рубрикатор глубиной более 3-х рубрик, т.к. по правилам создания интерфейсов, любой пользователь интерфейса должен получить нужную ему информацию максимум в 3 клика.
  6. Не ленитесь создать красивую, удобную, и информативную структуру характеристик для товарных групп, и, при добавлении товаров, полностью заполнять данные этих характеристик. Самая популярная ошибка заполнения базы данных - создание одной характеристики "Характеристики" с типом данных "большой текст", и описание всех характеристик в ней. Это грубейшая ошибка, вот её основные последствия:
    • посетитель сайта не сможет отфильтровать с помощью специальной формы нужные ему товары, а значит посетителю будет не удобно искать нужные товары, и он может уйти к конкурентам.
    • оператор не сможет постоянно вводить информацию о товарах, и их характеристиках в одинаковом порядке, использовать одинаковые визуальные разделители характеристик, стили описания, в то время, как при вводе каждой характеристики в отдельное поле по заданной структуре, вывод на сайте будет генерироваться программно, с заданными стилями и в нужном порядке.
  7. Если у товаров есть характеристика, обозначающая другие товары (напр. запчасти для автомобиля, или флеш-карта для мобильного телефона), то необходимо завести характеристику типа «другой товар» (напр. «комплектующие», или «опции»), и при добавлении товаров, связать их. Это необходимо для более удобной навигации пользователя по сайту, т.к. при просмотре информации о товаре, посетитель без проблем сможет посмотреть подробную информацию о каждом элементе комплектации товара.
  8. Для аналогичных характеристик (напр. Габаритные размеры – Длина, Ширина, Высота) делайте для каждой характеристики отдельные записи. Это нужно для удобства навигации посетителей по сайту, а так же для поиска товаров по заданным характеристикам.
  9. Если характеристика имеет единицу измерения (напр. длина может иметь единицу измерения – мм, см, метры, дюймы), пишите сокращение этой единицы измерения через запятую, после наименования характеристики (напр. «Длина, мм.», или «Мощьность двигателя, л.с.»). На сайте такие наименования будут автоматически распознаны программой, и наименование единицы измерения будет переноситься в конец, после данных характеристики (напр. «Длина: 100 мм.», или «Мощьность двигателя: 50 л.с.»).
  10. Если характеристика имеет тип данных «число», при заполнении значений этой характеристики можно использовать символы «точка» или «запятая» для разделения целой части числа от дробной.
  11. Для диапазонов значений используйте тип данных «текст».
